# Claude Codeへの引き継ぎメモ

このファイルは、claude.ai(チャット)での開発をClaude Code(ターミナル)に引き継ぐための
状況共有メモです。Claude Codeを起動したら、最初にこの内容を伝えてください
(例: 「HANDOFF.mdを読んで、現状を把握してください」)。

## プロジェクト概要

営業・CS支援業務を一気通貫管理するWebアプリのMVP。
Next.js(App Router) + TypeScript + Supabase + OpenAI(GPT API)構成。

## 現在動いている機能

* 企業一覧(検索・業種/ステータス絞り込み)・企業新規登録(担当者選択あり)
* 企業詳細(基本情報編集・AI商談準備生成・案件一覧/新規作成)
* ユーザー管理(`/users`、担当者選択肢のための最低限の実装)
* 文字起こし入力(`/transcripts/new`、案件を選んでテキスト貼り付け)
* 商談FB一括生成(`/feedback/generate`、議事録→FB→BANT→温度感→受注確度を連鎖生成し案件に反映)
* ダッシュボード(`/`、案件数・対応中・今月受注・スコア要確認の集計)
* ログイン画面(`/login`、Googleログインボタンのみ配線。**セッション保護は未実装**)
* GPTモックモード(`.env.local`に`OPENAI_MOCK=true`で、実際に課金せず動作確認できる)

## 既知の未実装・要対応事項

1. **認証・セッション制御が未実装** — ログインボタンはあるが、ログインしなくても
   全画面にアクセスできてしまう。Next.js middlewareでの保護が必要。
2. **ロール別の権限制御が未実装** — DB上はロール(admin/manager/cs/sales/support)を
   持っているが、画面側で表示・編集を制限する処理がまだない。
3. **案件ステータス・フェーズの手動変更UIがない** — `deals.stage`はDB上にはあるが、
   画面から変更する手段がまだない(現状は作成時のデフォルト値のまま)。
4. **次回アクション(action_items)の登録UIがない** — ダッシュボードには表示欄のみ用意。
5. **Google Docs出力・Google Sheets反映が未実装** — 元の要件にあった機能。
6. **生成履歴の独立した検索画面がない**。
7. **管理者設定画面が未実装**。
8. **OpenAIのクレジット/請求設定は本番運用前に必須**(`OPENAI_MOCK=false`に切り替えて確認すること)。

## 開発環境

* Supabaseプロジェクト: 作成済み、`supabase/migrations/0001_init.sql`を実行済み
* `.env.local`: 作成済み(Supabase・OpenAIのキーが入っている。Claude Codeに
  キーの中身を貼る必要はない。ファイルの存在と`OPENAI_MOCK`の値だけ確認すればよい)
* `npm install`・`npm run dev`は動作確認済み
