# バックログ・トレーサビリティ台帳

外部のチケット管理ツール(Linear/Jira等)が未接続のため、このリポジトリ内で要件・不具合の追跡を行う暫定台帳。

## 使い方

- 新しい要件が決まったら「要件バックログ」に1行追加し、`docs/requirements.md`のFR/NFR番号と対応付ける。
- 不具合を発見したら「不具合ログ」に発見時点で1行追加し、修正後にステータス・修正コミットを更新する。
- ステータスは `未着手` / `対応中` / `完了` / `保留` の4種類のみを使う。
- 何を優先するか(P0=即対応〜P2=時間があれば)は都度チームで合意し、ここに書いた優先度は絶対ではない。
